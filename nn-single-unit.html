ニューラルネットワークの最小単位

fit() の中で境界が動く

rkskmt

ここまでの道具

  • 勾配降下法：傾きの逆方向に少しずつ進んで最小値を探す（最適化の回）
  • 回帰の fit() は自作済み：体長→重さの直線を勾配降下で当て、sklearn と一致した（前回）
  • 残るは分類：ロジスティック回帰の fit() は、決定境界の回からブラックボックスのまま

今回の問い

同じ道具で、分類の fit() も開けられるか？ — 開けると、そこにニューラルネットワークの最小単位が現れる

最小単位の構造

入力から出力まで、たった2ステップ

Step 1：アフィン変換（affine transformation）

\[z = \alpha_1 x_1 + \alpha_2 x_2 + b\]

  • \(x_1, x_2\)：入力（輝度、重さ）
  • \(\alpha_1, \alpha_2\)：重み（どの特徴量をどれだけ重視するか）
  • \(b\)：バイアス（全体のオフセット）
  • \(z\)：スカラー1個

幾何的な意味

  • 2次元の点を \(\alpha\) 方向に射影して1次元に落とす
  • \(b\) で位置を調整する

Step 2：シグモイドで「確率」にする

\[p = \sigma(z) = \frac{1}{1 + e^{-z}}\]

  • 実数全体 \((-\infty, +\infty)\) を 0〜1 の範囲に押し込む
  • \(p \geq 0.5\) → Salmon、\(p < 0.5\) → Sea bass と判定
  • \(z = 0\)（＝決定境界の上）のとき、ちょうど \(p = 0.5\)

実はロジスティック回帰

  • 「アフィン変換 → シグモイド」 — この2ステップがニューラルネットワークの最小単位（1個のニューロン）
  • そして、これはロジスティック回帰そのもの
  • sklearn の LogisticRegression が中でやっているのもこの計算

今回の作戦

  • NN は、この最小単位の組み合わせでできている
  • だから、まず1個だけを完全に理解する

学習 ＝ 境界を動かすこと

何を最小化するか：損失関数

  • 「いまのパラメータがどれだけダメか」を1つの数字にする

\[L = -\frac{1}{N}\sum_{i} \left[ y_i \log p_i + (1 - y_i) \log (1 - p_i) \right]\]

  • 交差エントロピー（cross entropy）と呼ぶ
  • 例：正解 \(y=1\)（Salmon）なのに \(p = 0.4\) しか出せない
    • \(L = -\log(0.4) \approx 0.92\)（大きい＝ダメ）
  • 自信を持って正解（\(p=0.99\)）なら \(-\log(0.99) \approx 0.01\)（小さい＝良い）

クイズ

知りたいのは正答率。なぜ正答率そのものを最大化しない？

  • 正答率はパラメータを少し動かしても階段状にしか変化しない → 微分が 0 か未定義
  • 勾配降下法が使えない。滑らかな代理として交差エントロピーを最小化する

勾配が驚くほど簡単になる

  • 損失 \(L\) を \(z\) で微分すると、連鎖律（chain rule）で

\[\frac{\partial L}{\partial z} = \frac{\partial L}{\partial p} \cdot \frac{\partial p}{\partial z} = \left(-\frac{1}{p}\right) \cdot p(1-p) = p - 1 \quad (\text{正解 } y = 1 \text{ のとき})\]

  • 正解が 0 のときは \(p - 0\)。つまり常に

\[\frac{\partial L}{\partial z} = p - y\]

  • 各パラメータの更新式（学習率 \(\eta\)）：

\[\alpha \leftarrow \alpha - \eta\,(p - y)\,x, \qquad b \leftarrow b - \eta\,(p - y)\]

ノート

シグモイドの微分 \(p(1-p)\) と交差エントロピーの微分 \(-1/p\) が打ち消し合うようにできている。 「予測と正解の差 \((p-y)\) を、入力の大きさに応じて配る」だけ。

# Salmon, Sea bass, Salmon, ... と交互に並べ替えておく
order = np.empty(80, dtype=int)
order[0::2] = np.arange(0, 40)
order[1::2] = np.arange(40, 80)
X_norm = X_norm[order]
y      = y[order]
def sigmoid(z):
    return 1 / (1 + np.exp(-z))

def forward(X, alpha, b):
    """アフィン変換 → シグモイド"""
    return sigmoid(X @ alpha + b)

def loss_fn(y_true, p):
    """交差エントロピー"""
    eps = 1e-10
    p = np.clip(p, eps, 1 - eps)
    return -np.mean(y_true * np.log(p) + (1 - y_true) * np.log(1 - p))

def accuracy(y_true, p):
    return np.mean((p >= 0.5) == y_true)
  • forward が予測、loss_fn が採点。勾配は \((p-y)x\) なので関数すら要らない

魚を1匹ずつ見せて学習する

alpha = np.array([0.0, 0.0])   # 重みはゼロからスタート
b  = 0.0
lr = 0.5                       # 学習率

for i in range(10):
    xi, yi = X_norm[i], y[i]
    p_i   = sigmoid(xi @ alpha + b)   # Forward：1匹だけ予測
    delta = p_i - yi                  # 予測と正解の差
    alpha = alpha - lr * delta * xi   # 勾配降下で更新
    b     = b  - lr * delta

    p_all = forward(X_norm, alpha, b)
    print(f"{i+1}匹目 | Loss: {loss_fn(y, p_all):.4f} | 正答率: {accuracy(y, p_all):.1%} "
          f"| α=[{alpha[0]:6.3f}, {alpha[1]:6.3f}] | b={b:6.3f}")

ノート

sklearn の fit() がやっているのは、本質的にこのループ（の高速・安定版）。 ブラックボックスが1つ開いた。

まとめ

  • NN の最小単位 ＝ アフィン変換 → シグモイド ＝ ロジスティック回帰
  • 学習 ＝ 交差エントロピーを勾配降下法で最小化
    • 勾配は \((p - y) x\) — シグモイド＋交差エントロピーの組み合わせの妙
  • fit() の中身は「予測 → 差を測る → 境界を少し動かす」の繰り返しだった
