# Troubleshooting

## Which CSS file to edit

All slide CSS ships from the **cleanslidekit extension**: edit
`_extensions/rkskmt/cleanslidekit/custom.css` (or `theme.scss` for theme-level
variables). Do **not** edit the root-level `custom.css` — it is a build copy
that `quarto render` regenerates from the extension (`format-resources`), so
edits there silently vanish on the next render.

## Two CSS layers — Pandoc vs reveal

Code-block HTML is generated by **Pandoc**, which injects its own styles; slide
layout/typography belongs to the **reveal theme** layer. Overriding each needs
different selectors (both go in the extension `custom.css`):

- **Reveal layer** (headings, body text, tables, callouts): prefix with
  `.reveal` (e.g. `.reveal h2 { … }`), as the existing rules in `custom.css` do.
- **Pandoc layer** (code blocks): bare selectors + `!important`:

| What | Selector |
|---|---|
| Line number counter | `pre.numberSource code > span > a:first-child::before` |
| Span positioning for line numbers | `pre.numberSource code > span` |
| Pre margin/padding for line numbers | `pre.numberSource` |
| Language class on pre | `pre.shell`, `pre.python`, etc. |
| Syntax highlight spans | `code span.kw`, `code span.im`, etc. |

## Stale cache after changes

If changes to `_metadata.yaml`, the extension CSS/JS, or `.qmd` files don't
appear, clear both `_site` and `.quarto` and restart preview:

```bash
rm -rf _site .quarto && quarto preview --port 4321
```

The browser may also cache `custom.css` — hard-reload (Ctrl+Shift+R) when a CSS
change still doesn't show.
