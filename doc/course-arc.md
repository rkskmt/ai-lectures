# 講義全体のアーク設計

講義シリーズ全体の幹と、未執筆回の仕様をまとめる。設計の根拠は [engaging-lecture-design.md](engaging-lecture-design.md)（IP3200 で効果を確認した手法）。回を追加・並べ替えたら、このドキュメントと `index.qmd`・`_quarto.yml` を同時に更新する。

## 全体構成：全14回

最終回は**手書き数字分類を PyTorch で自作**（MLP で完結。CNN は導入しない — アウトロで「この先」として1枚触れるのみ）。この枠に収めるための決定（2026-06）：

- **Newton法は参考資料に降格**（`newton-method-appendix.qmd`）。本編の最適化回は勾配降下法に専念。NN 幹で使うのは GD だけ。
- **「現代のAI」（CNN・Attention）の講義回は廃止**。最終回が現代 AI の実演を兼ねる。Attention/LLM は概論の歴史パートで触れる。
- クラスタリング・PCA は2回のまま残す（魚の物語で完成済み、14回に収まる）。

回割り：導入1（概論）＋ 分類問題と特徴量5 ＋ fit() の中身8（勾配降下法／線形回帰と正則化／NN最小単位／変換を重ねる／バックプロパゲーション／NumPy実装／PyTorch入門／手書き数字プロジェクト）。

## 講義全体のコールドオープン

- `ai-introduction.qmd` の冒頭で、**直線では絶対に分けられないデータを機械が自力で分けた図**（NN の曲がった決定境界）を見せ、続けて**手書き数字の格子図**で最終回の成果物を予告する。
- 宣言：「この講義の最後までに、この仕組みを**中身から全部**組み立てる。ブラックボックスは残さない」
- 種明かしの場所：NN 多層回（曲がる境界の正体＝ReLU の折れ線）と実装回（自分のコードで同じ図を再現）、最終回（手書き数字が読めるようになる）。

## 設計メモ（執筆時の仕様、全回実装済み）

執筆完了後も、各回を改訂するときの設計意図のリファレンスとして残す。

### 線形回帰・多項式回帰

- 題材は実データ1本（engaging-lecture-design.md の実データ主義。気象庁の気温データ等、採用前にスパイク・形を実測で検証）。
- 構成：直線を**目で**引く → 当てずっぽうの限界 → 損失（二乗誤差）を定義 → 勾配降下で fit（最適化回の道具をそのまま使う）→ `sklearn` と答え合わせ。
- 多項式次数を上げて過学習を**起こして見せる**（クイズ：「次数を上げたら精度はどうなると思う？」→ めくり）→ 正則化。過学習と汎化の回の train/test・ギャップの読み方がここで再登場する。
- 過学習は3段構成のスレッドになっている：2次元決定境界の回で「あやしい満点」を見せる（引き）→ 過学習と汎化の回で正体を暴く（本編）→ この回で正則化という対抗手段を与える（解決）。書くときはこの接続を維持する。

### バックプロパゲーション

- コールドオープン：「多層になった瞬間、勾配の計算が大変になる。手で全部微分するしかない？」
- 2層・パラメータ数個のネットで**手計算1往復**（forward の数値 → loss → 逆向きに δ）。数値は必ずコードで検証してから本文に書く。
- `nn-multilayer.qmd` の「誤差が逆向きに流れる」スライドが伏線。ここで連鎖律として正体を明かす。

### 実装（NumPy）

- ゴール明示：「コールドオープンの図を、ライブラリなしの自分のコードで再現する」。
- `nn-single-unit.qmd` の forward / backward 関数を多層に一般化するだけ、という見せ方（新しい概念ゼロ、組み立てのみ）。

### PyTorch 入門

- 構成：NumPy 実装と**同じネットを並べて**書く（描き比べの装置）。`loss.backward()` 一発と backprop 手計算の結果が一致する瞬間が山場。
- 依存：`environment.yml` に pytorch-cpu / torchvision を追加済み。実際の `ai` Conda 環境にもインストール済み（2026-06）。

### 手書き数字分類プロジェクト（最終回）

- ゴール：概論の冒頭で予告した**手書き数字を読み取る機械**を、PyTorch の MLP で完成させる（コールドオープン回収③）。
- ネットは **MLP（全結合のみ）** — ここまでに組み立てた部品（アフィン変換・ReLU・ソフトマックス・交差エントロピー・勾配降下・backprop）だけで動くことが「ブラックボックスは残さない」の証明になる。CNN は**導入しない**。
- データ：MNIST（torchvision）。授業前にダウンロード手順を確認しておく。授業環境で落とせない場合の代替は `sklearn.datasets.load_digits`（8×8、概論の図と同じ）。
- 演出：学生自身が紙に書いた数字を読ませるデモができれば最強（撮影 → 28×28 に縮小 → 推論）。事前に実データで検証してから採用（直感は外れる、の原則）。
- 締め：「この先」1枚 — 画像が大きくなったら？（CNN）、入力が文章だったら？（Attention/LLM）— を**名前の紹介だけ**して講義全体を閉じる。
