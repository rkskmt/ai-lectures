設計：ループを行列に畳む

  • 中間の変換を16個並べる。1個ずつ書くと16本の式 — 行列にまとめる
記号 形 意味
\(W_1\) \((2, 16)\) 中間16変換ぶんの重み（列が1変換）
\(\mathbf{b}_1\) \((16,)\) 中間のバイアス
\(\mathbf{w}_2\) \((16,)\) 出力層の重み（多数決の票）
\(c_2\) スカラー 出力層のバイアス
  • 200点まとめて流すのも行列のまま：\(Z = X W_1 + \mathbf{b}_1\) は \((200, 16)\)
  • 「変換16個 × データ200点」がたった1回の行列積

Forward：4行

def forward(X, W1, b1, w2, c2):
    Z = X @ W1 + b1          # アフィン変換（全変換まとめて）
    H = np.maximum(0, Z)     # ReLU（折る）
    u = H @ w2 + c2          # 出力層（多数決）
    p = 1 / (1 + np.exp(-u)) # シグモイド（確率に）
    return Z, H, u, p
  • 最小単位の回の forward と見比べると、スカラーが行列になっただけ
  • \(Z, H\) も返すのは、復路（backprop）で使うから（前回の伏線）

Backward：前回の漸化式をそのまま書く

def backward(X, y, Z, H, w2, p):
    N = len(X)
    delta_out = (p - y) / N                    # 出力層の誤差（p - y）
    g_w2 = H.T @ delta_out                     # ∂L/∂w2 = δ_out × 入力(H)
    g_c2 = delta_out.sum()                     # ∂L/∂c2 = δ_out
    D_mid = np.outer(delta_out, w2) * (Z > 0)  # δ_mid = δ_out・w2・ReLU'(Z)
    g_W1 = X.T @ D_mid                         # ∂L/∂W1 = δ_mid × 入力(X)
    g_b1 = D_mid.sum(axis=0)                   # ∂L/∂b1 = δ_mid
    return g_W1, g_b1, g_w2, g_c2
  • 1行ずつ、前回の手計算と完全に対応している
    • (p - y)：シグモイド＋交差エントロピーの打ち消し
    • * (Z > 0)：\(\mathrm{ReLU}'\) は 1 か 0
    • 「\(\delta\) × その層への入力」が各勾配

学習ループ：あとは勾配降下法

rng = np.random.default_rng(0)
hidden = 16
W1 = rng.normal(0, 1.0, (2, hidden))   # ランダム初期化（対称性を破る）
b1 = np.zeros(hidden)
w2 = rng.normal(0, 1.0, hidden)
c2 = 0.0
lr = 0.5

for i in range(1, 4001):
    Z, H, u, p = forward(X, W1, b1, w2, c2)
    g_W1, g_b1, g_w2, g_c2 = backward(X, y, Z, H, w2, p)
    W1 -= lr * g_W1
    b1 -= lr * g_b1
    w2 -= lr * g_w2
    c2 -= lr * g_c2

    if i in (1, 100, 500, 2000, 4000):
        eps  = 1e-10
        loss = -np.mean(y * np.log(np.clip(p, eps, 1)) +
                        (1 - y) * np.log(np.clip(1 - p, eps, 1)))
        acc  = np.mean((p >= 0.5) == y)
        print(f"更新 {i:5d}回 | Loss: {loss:.4f} | 正答率: {acc:.1%}")

約束の回収

講義の初回、「数式も規則も誰も教えていないのに機械が見つけた境界」として見せた図。 いまこの図は、1行残らず自分が書いたコードから出ている。中身に未知の部品はもうない。

折り目の数で遊ぶ

  • 中間2個では折り目が足りない。16個なら滑らかに曲がる — 「変換を重ねる」の回の結論を、自作コードで追試できた
  • hidden や学習率をいじって遊んでみること。壊れても全部自分のコードなので、原因も自分で追える

まとめ

  • NumPy の行列演算で、forward 4行・backward 6行の多層ネットワークができた
  • 中身は既習の部品のみ：アフィン変換・ReLU・\((p-y)\)・\(\delta\) の漸化式・勾配降下法
  • 初回のコールドオープンの図を、自分のコードで再現した
