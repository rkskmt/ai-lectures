本編の振り返りと今回の問題

本編でやったこと

本編では \(f(x) = x^2 - 4x + 3\) という2次関数で、勾配降下法（と参考資料の Newton法）を学びました。

この関数は谷が1つだけのシンプルな形でした。 どこから始めても、必ず同じ最小値 \(x = 2\) にたどり着きました。

本編の結論

  • 勾配降下法：何回も反復して少しずつ最小値に近づく
  • Newton法：2次微分も使って、一気に最小値にジャンプ
  • どちらも同じ答えにたどり着いた

今回の疑問

でも、こんな疑問が浮かびます：

「谷が2つ以上ある関数だったら、どうなるの？」

実際の機械学習の問題では、谷が1つだけということはほとんどありません。 パラメータが多くなると、たくさんの谷が存在します。

今回は「谷が2つある関数」で、Newton法を使ったときに何が起こるかを体験しましょう。

谷が2つある関数を見てみよう

関数の定義

今回使う関数はこれです：

\[f(x) = 0.1x^4 - x^3 + 2x^2 + 1\]

本編と同じように、微分を手で計算しておきます。

1次微分（傾き）: \[f'(x) = 0.4x^3 - 3x^2 + 4x\]

2次微分（曲がり具合）: \[f''(x) = 1.2x^2 - 6x + 4\]

Pythonでの定義

import numpy as np
import matplotlib.pyplot as plt
import japanize_matplotlib

# 4次関数とその微分
def f_poly(x):
    return 0.1*x**4 - x**3 + 2*x**2 + 1

def f_poly_prime(x):
    return 0.4*x**3 - 3*x**2 + 4*x

def f_poly_double_prime(x):
    return 1.2*x**2 - 6*x + 4

本編の2次関数は谷が1つだけでしたが、今回は2つあります。

さて、Newton法を使ったら、どちらの谷に行くのでしょうか？

実験：初期値を変えてNewton法を実行してみよう

作戦：式は同じ、スタート地点だけ変える

Newton法（参考資料）の更新式を思い出しましょう：

\[x_{\text{new}} = x - \frac{f'(x)}{f''(x)}\]

同じ式を使って、スタート地点だけ変えて2回実験します。

実験1：\(x_0 = 0.5\) からスタート（左の谷の近く）

手計算で追いかけてみよう

反復1

現在の位置: \(x = 0.5\)

1次微分（傾き）を計算: \[f'(0.5) = 0.4 \times 0.5^3 - 3 \times 0.5^2 + 4 \times 0.5\] \[= 0.05 - 0.75 + 2.0 = 1.3\]

2次微分（曲がり具合）を計算: \[f''(0.5) = 1.2 \times 0.5^2 - 6 \times 0.5 + 4\] \[= 0.3 - 3.0 + 4.0 = 1.3\]

更新: \[x_{\text{new}} = 0.5 - \frac{1.3}{1.3} = 0.5 - 1.0 = -0.5\]

反復2

現在の位置: \(x = -0.5\)（谷底を飛び越した。でも大丈夫）

1次微分: \[f'(-0.5) = 0.4 \times (-0.125) - 3 \times 0.25 + 4 \times (-0.5) = -0.05 - 0.75 - 2.0 = -2.8\]

2次微分: \[f''(-0.5) = 1.2 \times 0.25 - 6 \times (-0.5) + 4 = 0.3 + 3.0 + 4.0 = 7.3\]

更新: \[x_{\text{new}} = -0.5 - \frac{-2.8}{7.3} = -0.5 + 0.384 = -0.116\]

反復3

現在の位置: \(x = -0.116\)

\[f'(-0.116) \approx -0.51, \qquad f''(-0.116) \approx 4.71\] \[x_{\text{new}} = -0.116 + 0.108 = -0.009\]

反復4

\(f'(-0.009)\) はほぼゼロ。収束しました！

実験1の結果

\(x_0 = 0.5\) からスタート → \(x \approx 0\)（左の谷の底）に収束、\(f(0) = 1\)

念のため \(f''(0) = 4 > 0\) — ちゃんと谷です。

2つの実験を並べてみよう

たどり着いた谷が違う

まったく同じNewton法を使ったのに：

スタート地点 到着地点 f(x)の値 そこは何？
実験1 \(x_0 = 0.5\) \(x \approx\) -0.00 1.00 谷（左）
実験2 \(x_0 = 5.5\) \(x \approx\) 5.77 -13.67 谷（右・もっと深い）
  • どちらも正真正銘の谷（\(f''>0\)）。でも深さがまったく違う
  • 実験1の答えは「その周辺では最適、全体では2番手」— そして使った本人は、もっと深い谷の存在に気づけない

スタート地点（初期値）が違うだけで、たどり着く谷が変わってしまう。

この現象に名前をつけよう

ローカルミニマムとグローバルミニマム

いま体験した現象には、ちゃんと名前がついています。

ローカルミニマム（局所的最小値, local minimum）: 「その周辺では一番低い」けど、関数全体で見るともっと低い場所がある点。 実験1で到達した左の谷（x = 0）がこれです。

グローバルミニマム（大域的最小値, global minimum）: 関数全体で本当に一番低い点。 実験2で到達した右の谷（x ≈ 5.77）がこれです。

たとえ話：山の中で一番低い場所を探す

あなたは目隠しをして山の中にいます。足元の傾きだけを頼りに、一番低い場所を探します。

  • 近くの谷に降りることはできる → ローカルミニマム
  • でも、山の向こうにもっと深い谷があるかもしれない → グローバルミニマム
  • 目隠しをしているので、今いる谷が一番深いかどうかわからない！

Newton法も勾配降下法も、この「目隠し」と同じ状況です。

実験3：丘の中腹 \(x_0 = 1.5\) から始めると…

  • 同じ更新式を回すと：\(1.5 \rightarrow 1.761 \rightarrow 1.735 \rightarrow\) 停止（\(f' \approx 0\)）
  • 左に下り坂があるのに、坂を登って丘のてっぺんで止まった

なぜ登るのか：

  • 一歩の向きは \(-f'/f''\)。ここで \(f'(1.5) = +0.6\)（右上り坂）、\(f''(1.5) = -2.3\)（上に凸）
  • 符号が打ち消し合って、一歩は \(+0.26\) — 傾きと同じ向き、つまり登る向きになる
  • Newton法が探しているのは「\(f' = 0\) の点」だけ。そこが山か谷かは見ていない
  • 停止条件（\(|f'|\) がほぼゼロ）は山頂でも満たされる → 山で本当に止まる

\(f''\) チェックは「止まった後の検品」

収束したら \(f''(x)\) を確認：正なら谷、負なら山 — 山なら結果を捨てる。 山か谷かの区別はできるが、それは止まってからの話。 走っている最中のNewton法には、登りを避けるブレーキがついていない。 （実は後付けできる — 引力圏とカタパルトを見たあとで「修理」する）

なぜこうなるのか：初期値と「引力圏」

Newton法は「近くの傾きゼロ」に落ちる

Newton法は、現在地の近くにある「傾きゼロの点」を見つけるアルゴリズムです。 遠くのことは知りませんし、そこが谷か山頂かも気にしません。

だから、だいたいは：

  • 左の谷の近くからスタート → 左の谷（ローカルミニマム）に落ちる（実験1）
  • 右の谷の近くからスタート → 右の谷（グローバルミニマム）に落ちる（実験2）
  • 丘の上からスタート → 丘のてっぺんに登って止まる（実験3）

本当に「だいたい」で済むのか、初期値を200個並べて全部試してみましょう。

引力圏（Basin of Attraction）を可視化しよう

「どの初期値からスタートすると、どの谷に落ちるか」を全部調べてみましょう。

コードを表示 
def newton_optimize(f, fp, fpp, x0, max_iter=50):
    """Newton法を実行して結果を返す"""
    x = x0
    for i in range(max_iter):
        grad = fp(x)
        hess = fpp(x)
        if abs(grad) < 1e-6:
            return x, f(x), True
        if abs(hess) < 1e-10:
            return x, f(x), False
        x = x - grad / hess
    return x, f(x), False

# 初期値を -1.5 から 7.5 まで細かく変えて、全部試す
x0_dense = np.linspace(-1.5, 7.5, 200)
convergence_points = []
convergence_values = []

for x0 in x0_dense:
    x_final, f_final, _ = newton_optimize(f_poly, f_poly_prime, f_poly_double_prime, x0)
    convergence_points.append(x_final)
    convergence_values.append(f_final)

convergence_points = np.array(convergence_points)
convergence_values = np.array(convergence_values)

  • 縦に1本選んで読む：「x0 = −1 から出発 → 点の高さは 0 → 左の谷に着いた」

引力圏（Basin of Attraction）とは

各収束先には「引力圏」があります。その範囲からスタートすると、そこに吸い込まれる領域です。 この実験では、素朴な予想より意地悪な結果になりました：

  • 赤い帯 → ローカルミニマム、緑の帯 → グローバルミニマム、ここまでは予想どおり
  • 橙の帯 → 山のてっぺん（極大）に到着して止まる。Newton法が探しているのは「傾きゼロの場所」なので、谷と山頂を区別できない
  • 帯はきれいな2分割ではない：グローバルミニマムのすぐ手前（x0 ≈ 3.6〜4.2）から出発しても、左の谷へ飛ばされる
  • 初期値が少し変わるだけで行き先がガラッと変わる — 1回の実行を信用してはいけない理由がこれ

なぜ「すぐ手前の帯」が左へ飛ぶのか

x0 = 4.0 から始めたときの最初の1歩を解剖してみます。

  • Newton法の1歩は「いまの場所に放物線を当てはめて、その頂点へジャンプ」（\(x - f'/f''\) の意味）
  • ところが x0=4 は上に凸の区間（2つの変曲点 x≈0.79〜4.21 の間）の右端 — 当てはめた放物線が逆さまになり、頂点は左の遠く（x=−4）にある
  • しかも変曲点の近くでは \(f'' \approx 0\) なので、割り算 \(f'/f''\) が爆発して飛距離が伸びる
  • 着地した x=−4 は左の谷の素直な（下に凸の）区間 → あとはおとなしく x=0 に収束
  • 細い緑の帯（x0≈0.85）は同じ現象の逆向き：左の変曲点のすぐ内側から +6.3 ジャンプして右の谷へ

重要

大ジャンプできるのがNewton法の強みで、放物線近似が嘘をつく場所では同じ大ジャンプが暴発する。 勾配降下法なら「傾きの逆に少しずつ」なので、x0=4 からは素直に右の谷へ降りる。

— では、この壊れ方は直せないのか？　直せます。次で修理しましょう。

修理する：ワープの前に「検品」を入れる

故障の原因は「無条件ワープ」

ここまでに見た故障は2種類：

  • 山頂で止まる（実験3）— 上に凸の場所では、一歩が登る向きになる
  • 反対側へ吹っ飛ぶ（カタパルト）— 変曲点の近くでは、一歩が大きすぎる

原因はどちらも同じ。Newton法が、当てはめた放物線の頂点へ検査なしでワープしていることです。

ヒント

修理方針：提案された一歩を、2つの検品に通してから踏む。

検査1：その方向、本当に「下り」？

  • Newton法の一歩 \(d = -f'/f''\) は提案として扱う
  • 踏む前にチェック：\(f' \cdot d \geq 0\) なら、提案は傾きと同じ向き＝登りなので却下
  • 却下したら、確実に下る最急降下方向 \(d = -f'\) に差し替える
  • 登り提案が出るのは、ちょうど \(f'' < 0\)（上に凸）の場所 — 実験3とカタパルトの現場そのもの

実験3の現場検証：\(x = 1.5\) では提案 \(d = +0.26\)。 \(f' \cdot d = 0.6 \times 0.26 > 0\) → 登りなので却下 → \(d = -f' = -0.6\)（左の下りへ）

検査2：その一歩、実測で下がってる？

  • 向きが正しくても、歩幅が大きすぎると谷を飛び越えて高い場所に着くことがある
  • そこで、踏む前に着地点の本物の \(f\) を実測する：
    • 十分下がっていれば採用
    • 下がっていなければ歩幅を半分にしてやり直し（バックトラッキング）
  • 放物線の「予言」ではなく、実測で確認してからしか進まない

修理版Newton法（検査1・2入り）

def newton_safe(f, fp, fpp, x0, max_iter=100):
    """Newton法＋検査1（降下方向）＋検査2（実測バックトラッキング）"""
    x = x0
    for _ in range(max_iter):
        g = fp(x)
        if abs(g) < 1e-8:                  # 傾きほぼゼロ → 停止
            break
        h = fpp(x)
        d = -g / h if h != 0 else -g       # Newton法の「提案」
        if g * d >= 0:                     # 検査1：登りなら却下して
            d = -g                         #   最急降下に差し替え
        alpha = 1.0
        while f(x + alpha * d) > f(x) - 1e-4 * alpha * abs(g * d):
            alpha /= 2                     # 検査2：実測で下がるまで歩幅半分
        x = x + alpha * d
    return x, f(x)

# 故障していた2つの初期値で試運転
for x0 in [1.5, 4.0]:
    x_fin, f_fin = newton_safe(f_poly, f_poly_prime, f_poly_double_prime, x0)
    print(f"x0 = {x0} → x = {x_fin:.4f}, f(x) = {f_fin:.4f}")
x0 = 1.5 → x = -0.0000, f(x) = 1.0000
x0 = 4.0 → x = 5.7656, f(x) = -13.6726

  • カタパルトだった \(x_0 = 4\) は、なんと右の谷（グローバルミニマム）へ — 暴発も消えた

  • scipy などの実用ソルバーは、この種の検品を毎歩組み込み済み：
    • BFGS：常に「下り向きの提案」しか出ないよう作った近似 \(f''\) を使う
    • Newton-CG：負の曲率（上に凸）を検出した瞬間に計算を打ち切り、勾配方向へ切り替え
    • 信頼領域法：放物線を「信じる半径」の中でだけ使う
  • 実測：\(x_0 = 1.5\) から scipy.optimize.minimize を呼ぶと、BFGS / Newton-CG / trust-exact のすべてが山頂に登らず \(x=0\) の谷に降りる
  • 流儀は違っても思想は同じ：「\(f\) が下がらない一歩は踏まない」

ノート

授業で書いた素のNewton法が「欠陥品」なわけではない。教科書のNewton法はもともと「\(f'=0\) を解く」方法で、 最小化に使うための安全装置は別売り — ライブラリにはそれが標準装備されている、ということ。

対策：どうすればグローバルミニマムを見つけられるか

修理しても、直らないものがある

修理後の引力圏図をもう一度思い出すと：

  • 「山に登る」「吹っ飛ぶ」は直った
  • でも行き先は相変わらず2つある — どっちの谷に落ちるかは初期値しだいのまま
  • これはNewton法の故障ではなく、目隠しで坂を下ること自体の限界（目の前の谷しか見えない）

解決策はシンプル：何回もやって、一番良い結果を選ぶ！

方法1：手で初期値を変えて比較する

まず、3つの初期値から手計算で比較してみましょう。

\(x_0 = 0\) からスタート

\[f'(0) = 0.4 \times 0 - 3 \times 0 + 4 \times 0 = 0\]

\(f'(0) = 0\) なので、\(x = 0\) は既に極値です。\(f(0) = 1\)。

\(f''(0) = 1.2 \times 0 - 6 \times 0 + 4 = 4 > 0\) → 最小値（谷）。

\(x_0 = 2\) からスタート

\[f'(2) = 0.4 \times 8 - 3 \times 4 + 4 \times 2 = 3.2 - 12 + 8 = -0.8\] \[f''(2) = 1.2 \times 4 - 6 \times 2 + 4 = 4.8 - 12 + 4 = -3.2\] Newton法の提案は \(d = -\dfrac{-0.8}{-3.2} = -0.25\)（左へ）。 でも傾きは右下り（\(f' < 0\)）なので、左へ進むのは登り — 検査1が却下して \(d = -f' = +0.8\)（右へ）に差し替えます。

→ そのまま右の谷（グローバルミニマム）まで降りていきます。素のNewton法なら丘のてっぺん行きでした。

\(x_0 = 6\) からスタート

\[f'(6) = 0.4 \times 216 - 3 \times 36 + 4 \times 6 = 86.4 - 108 + 24 = 2.4\] \[f''(6) = 1.2 \times 36 - 6 \times 6 + 4 = 43.2 - 36 + 4 = 11.2\] \[x_{\text{new}} = 6 - \frac{2.4}{11.2} = 6 - 0.214 = 5.786\]

→ 右の谷（グローバルミニマム）の方向に進んでいます。

3つの結果を比較

方法 やり方 長所 短所

どちらも「何回も試して一番良いのを選ぶ」という同じ考え方です。

実践チェックリスト

最適化の4ステップ

最適化問題に取り組むときは、以下の手順を守りましょう。

ステップ1：関数を可視化する

まずグラフを描いて、谷がいくつあるか確認する。 谷が複数ありそうなら、ローカルミニマムに注意！

ステップ2：複数の初期値を用意する

最低でも3〜5個。余裕があれば10個以上。 グリッドサーチかランダムリスタートで用意する。

ステップ3：全部実行して、結果を比較する

すべての初期値からNewton法を実行し、\(f(x)\) が一番小さい結果を選ぶ。

ステップ4：2次微分で確認する

収束した点で \(f''(x) > 0\) なら谷（最小値）、\(f''(x) < 0\) なら山なので捨てる。 検査1・2入りの修理版ならまず山には止まらないが、確認は1行 — 保険として常に入れておく （素のNewton法を使ったときは必須）。

実装テンプレート

上の4ステップをまとめたコードです。コピーして使えます。

def robust_optimization(f, fp, fpp, x0_list, max_iter=100):
    """
    複数の初期値を使ったロバストな最適化
    
    Parameters:
    -----------
    f, fp, fpp : functions
        目的関数とその1次微分、2次微分
    x0_list : list
        初期値のリスト
    max_iter : int
        最大反復数
    
    Returns:
    --------
    best_x : float
        見つかった最良解
    all_results : list
        比較用のすべての結果
    """
    all_results = []

    for x0 in x0_list:
        x = x0

        for i in range(max_iter):
            grad = fp(x)
            if abs(grad) < 1e-6:
                break
            hess = fpp(x)
            d = -grad / hess if hess != 0 else -grad   # Newton法の提案
            if grad * d >= 0:                          # 検査1：登りなら
                d = -grad                              #   最急降下に差し替え
            alpha = 1.0
            while f(x + alpha * d) > f(x) - 1e-4 * alpha * abs(grad * d):
                alpha /= 2                             # 検査2：実測で確認
            x = x + alpha * d

        all_results.append({
            'x0': x0,
            'x_final': x,
            'f_final': f(x),
            'is_minimum': fpp(x) > 0   # 保険の検品（修理版なら通常すべて谷）
        })

    # 谷（最小値）だけを残して、一番良いのを選ぶ
    minima = [r for r in all_results if r['is_minimum']]
    if minima:
        best_result = min(minima, key=lambda r: r['f_final'])
    else:
        best_result = min(all_results, key=lambda r: r['f_final'])

    return best_result['x_final'], all_results

# 使用例：5つの初期値から試す

まとめ

今回学んだこと

ノート

  1. 谷が複数ある関数では、Newton法の結果が初期値に依存する
    • 左の谷の近く → 左の谷、右の谷の近く → 右の谷
    • 丘の上からだと山頂に登って止まることすらある（素のNewton法は \(f'=0\) を探しているだけ）
  2. ローカルミニマムとグローバルミニマム
    • ローカルミニマム = 周辺では一番低いが、全体では最低ではない
    • グローバルミニマム = 全体で本当に一番低い点
  3. 修理できる故障と、できない故障がある
    • 検査1（降下方向か）＋検査2（実測で下がったか）で「山に登る・吹っ飛ぶ」は直る — ライブラリは標準装備
    • どの谷に落ちるかだけは、どんなに修理しても初期値しだい
  4. 対策：複数の初期値から試して、一番良い結果を選ぶ
    • ランダムリスタートやグリッドサーチを使う
    • 2次微分の確認（\(f''>0\) なら谷）は保険として常に入れる
