前回の振り返りと今回の問題

前回やったこと

前回は \(f(x) = x^2 - 4x + 3\) という2次関数で、勾配降下法とNewton法を学びました。

この関数は谷が1つだけのシンプルな形でした。 どこから始めても、必ず同じ最小値 \(x = 2\) にたどり着きました。

前回の結論

  • 勾配降下法：何回も反復して少しずつ最小値に近づく
  • Newton法：2次微分も使って、一気に最小値にジャンプ
  • どちらも同じ答えにたどり着いた

今回の疑問

でも、こんな疑問が浮かびます：

「谷が2つ以上ある関数だったら、どうなるの？」

実際の機械学習の問題では、谷が1つだけということはほとんどありません。 パラメータが多くなると、たくさんの谷が存在します。

今回は「谷が2つある関数」で、Newton法を使ったときに何が起こるかを体験しましょう。

谷が2つある関数を見てみよう

関数の定義

今回使う関数はこれです：

\[f(x) = 0.1x^4 - x^3 + 2x^2 + 1\]

前回と同じように、微分を手で計算しておきます。

1次微分（傾き）: \[f'(x) = 0.4x^3 - 3x^2 + 4x\]

2次微分（曲がり具合）: \[f''(x) = 1.2x^2 - 6x + 4\]

Pythonでの定義

import numpy as np
import matplotlib.pyplot as plt

# 4次関数とその微分
def f_poly(x):
    return 0.1*x**4 - x**3 + 2*x**2 + 1

def f_poly_prime(x):
    return 0.4*x**3 - 3*x**2 + 4*x

def f_poly_double_prime(x):
    return 1.2*x**2 - 6*x + 4

まずグラフを描いてみよう

どんな形の関数なのか、まず目で見ることが大切です。

グラフを観察しよう

谷がいくつあるか、数えてみてください。

  • 左の谷（\(x \approx 1\) 付近）：浅い谷
  • 右の谷（\(x \approx 6\) 付近）：深い谷

前回の2次関数は谷が1つだけでしたが、今回は2つあります。

さて、Newton法を使ったら、どちらの谷に行くのでしょうか？

実験：初期値を変えてNewton法を実行してみよう

前回学んだNewton法の更新式を思い出しましょう：

\[x_{\text{new}} = x - \frac{f'(x)}{f''(x)}\]

同じ式を使って、スタート地点だけ変えて2回実験します。

その前に：\(f'(x) = 0\) は「谷」だけじゃない

実験に入る前に、1つだけ知っておくべきことがあります。

Newton法は \(f'(x) = 0\) となる点を探しますが、\(f'(x) = 0\) になるのは谷（最小値）だけではありません。山（最大値）でも \(f'(x) = 0\) です。

これは2次微分で見分けられます：

\[f''(x) > 0 \Rightarrow \text{谷（最小値）}\] \[f''(x) < 0 \Rightarrow \text{山（最大値）}\]

覚えておこう

収束したら \(f''(x)\) をチェック。正なら谷、負なら山。 山に止まった結果は捨てましょう。

実験1：\(x_0 = 1.5\) からスタート（左の谷の近く）

手計算で追いかけてみよう

反復1

現在の位置: \(x = 1.5\)

1次微分（傾き）を計算: \[f'(1.5) = 0.4 \times 1.5^3 - 3 \times 1.5^2 + 4 \times 1.5\] \[= 0.4 \times 3.375 - 3 \times 2.25 + 6.0\] \[= 1.35 - 6.75 + 6.0 = 0.6\]

2次微分（曲がり具合）を計算: \[f''(1.5) = 1.2 \times 1.5^2 - 6 \times 1.5 + 4\] \[= 1.2 \times 2.25 - 9.0 + 4.0\] \[= 2.7 - 9.0 + 4.0 = -2.3\]

更新: \[x_{\text{new}} = 1.5 - \frac{0.6}{-2.3} = 1.5 - (-0.261) = 1.5 + 0.261 = 1.761\]

反復2

現在の位置: \(x = 1.761\)

1次微分: \[f'(1.761) = 0.4 \times 1.761^3 - 3 \times 1.761^2 + 4 \times 1.761\] \[= 2.184 - 9.302 + 7.044 = -0.074\]

2次微分: \[f''(1.761) = 1.2 \times 1.761^2 - 6 \times 1.761 + 4\] \[= 3.720 - 10.566 + 4.0 = -2.846\]

更新: \[x_{\text{new}} = 1.761 - \frac{-0.074}{-2.846} = 1.761 - 0.026 = 1.735\]

反復3

\(f'(1.735)\) はほぼゼロに近づいています。収束しました！

実験1の結果

\(x_0 = 1.5\) からスタート → \(x \approx 1.74\) に収束

\(f(1.74) \approx 1.93\)

スタート地点（初期値）が違うだけで、たどり着く場所が変わってしまう。

この現象に名前をつけよう

ローカルミニマムとグローバルミニマム

いま体験した現象には、ちゃんと名前がついています。

ローカルミニマム（局所的最小値）: 「その周辺では一番低い」けど、関数全体で見るともっと低い場所がある点。 実験1で到達した左の谷がこれです。

グローバルミニマム（大域的最小値）: 関数全体で本当に一番低い点。 実験2で到達した右の谷がこれです。

たとえ話：山の中で一番低い場所を探す

あなたは目隠しをして山の中にいます。足元の傾きだけを頼りに、一番低い場所を探します。

  • 近くの谷に降りることはできる → ローカルミニマム
  • でも、山の向こうにもっと深い谷があるかもしれない → グローバルミニマム
  • 目隠しをしているので、今いる谷が一番深いかどうかわからない！

Newton法も勾配降下法も、この「目隠し」と同じ状況です。

なぜこうなるのか：初期値と「引力圏」

Newton法は「近くの谷」に落ちる

Newton法は、現在地の近くにある谷を見つけるアルゴリズムです。 遠くの谷のことは知りません。

だから：

  • 左の谷の近くからスタート → 左の谷（ローカルミニマム）に落ちる
  • 右の谷の近くからスタート → 右の谷（グローバルミニマム）に落ちる

引力圏（Basin of Attraction）を可視化しよう

「どの初期値からスタートすると、どの谷に落ちるか」を全部調べてみましょう。

コードを表示 
def newton_optimize(f, fp, fpp, x0, max_iter=50):
    """Newton法を実行して結果を返す"""
    x = x0
    for i in range(max_iter):
        grad = fp(x)
        hess = fpp(x)
        if abs(grad) < 1e-6:
            return x, f(x), True
        if abs(hess) < 1e-10:
            return x, f(x), False
        x = x - grad / hess
    return x, f(x), False

# 初期値を -1.5 から 7.5 まで細かく変えて、全部試す
x0_dense = np.linspace(-1.5, 7.5, 200)
convergence_points = []
convergence_values = []

for x0 in x0_dense:
    x_final, f_final, _ = newton_optimize(f_poly, f_poly_prime, f_poly_double_prime, x0)
    convergence_points.append(x_final)
    convergence_values.append(f_final)

convergence_points = np.array(convergence_points)
convergence_values = np.array(convergence_values)

引力圏（Basin of Attraction）とは

各谷には「引力圏」があります。その範囲内からスタートすると、その谷に吸い込まれるという領域です。

  • 赤い領域からスタート → ローカルミニマム（浅い谷）に落ちる
  • 緑の領域からスタート → グローバルミニマム（深い谷）に落ちる

初期値が少し変わるだけで、結果がガラッと変わる境界があることに注目してください。

対策：どうすればグローバルミニマムを見つけられるか

考え方：1回だけでは信用できない

ここまでの実験でわかったこと：

  • Newton法は「近くの谷」に落ちるだけ
  • 1回の実行では、それがローカルかグローバルかわからない

解決策はシンプル：何回もやって、一番良い結果を選ぶ！

方法1：手で初期値を変えて比較する

まず、3つの初期値から手計算で比較してみましょう。

\(x_0 = 0\) からスタート

\[f'(0) = 0.4 \times 0 - 3 \times 0 + 4 \times 0 = 0\]

\(f'(0) = 0\) なので、\(x = 0\) は既に極値です。\(f(0) = 1\)。

\(f''(0) = 1.2 \times 0 - 6 \times 0 + 4 = 4 > 0\) → 最小値（谷）。

\(x_0 = 2\) からスタート

\[f'(2) = 0.4 \times 8 - 3 \times 4 + 4 \times 2 = 3.2 - 12 + 8 = -0.8\] \[f''(2) = 1.2 \times 4 - 6 \times 2 + 4 = 4.8 - 12 + 4 = -3.2\] \[x_{\text{new}} = 2 - \frac{-0.8}{-3.2} = 2 - 0.25 = 1.75\]

→ 左の谷（ローカルミニマム）の方向に進んでいます。

\(x_0 = 6\) からスタート

\[f'(6) = 0.4 \times 216 - 3 \times 36 + 4 \times 6 = 86.4 - 108 + 24 = 2.4\] \[f''(6) = 1.2 \times 36 - 6 \times 6 + 4 = 43.2 - 36 + 4 = 11.2\] \[x_{\text{new}} = 6 - \frac{2.4}{11.2} = 6 - 0.214 = 5.786\]

→ 右の谷（グローバルミニマム）の方向に進んでいます。

3つの結果を比較

for x0 in starting_points:
    x_final, f_final, _ = newton_optimize(f_poly, f_poly_prime, f_poly_double_prime, x0)

    print(f"{x0:4.1f}  →  {xf:8.4f}  →  {fx:8.4f}  →{marker}")

ランダムリスタート vs グリッドサーチ

方法 やり方 長所 短所
ランダムリスタート 初期値をランダムに選ぶ 簡単、偏りが少ない 運に左右される
グリッドサーチ 初期値を等間隔に並べる 漏れが少ない 数が多いと大変

どちらも「何回も試して一番良いのを選ぶ」という同じ考え方です。

実践チェックリスト

最適化問題に取り組むときは、以下の手順を守りましょう。

ステップ1：関数を可視化する

まずグラフを描いて、谷がいくつあるか確認する。 谷が複数ありそうなら、ローカルミニマムに注意！

ステップ2：複数の初期値を用意する

最低でも3〜5個。余裕があれば10個以上。 グリッドサーチかランダムリスタートで用意する。

ステップ3：全部実行して、結果を比較する

すべての初期値からNewton法を実行し、\(f(x)\) が一番小さい結果を選ぶ。

ステップ4：2次微分で確認する

収束した点で \(f''(x) > 0\) なら谷（最小値）。 \(f''(x) < 0\) なら山（最大値）なので、その結果は捨てる。

実装テンプレート

上の4ステップをまとめたコードです。コピーして使えます。

def robust_optimization(f, fp, fpp, x0_list, max_iter=100):
    """
    複数の初期値を使ったロバストな最適化
    
    Parameters:
    -----------
    f, fp, fpp : functions
        目的関数とその1次微分、2次微分
    x0_list : list
        初期値のリスト
    max_iter : int
        最大反復数
    
    Returns:
    --------
    best_x : float
        見つかった最良解
    all_results : list
        比較用のすべての結果
    """
    all_results = []
    
    for x0 in x0_list:
        x = x0
        
        for i in range(max_iter):
            grad = fp(x)
            hess = fpp(x)
            
            if abs(grad) < 1e-6:
                break
            if abs(hess) < 1e-10:
                break
            
            x = x - grad / hess
        
        all_results.append({
            'x0': x0,
            'x_final': x,
            'f_final': f(x),
            'is_minimum': fpp(x) > 0
        })
    
    # 谷（最小値）だけを残して、一番良いのを選ぶ
    minima = [r for r in all_results if r['is_minimum']]
    if minima:
        best_result = min(minima, key=lambda r: r['f_final'])
    else:
        best_result = min(all_results, key=lambda r: r['f_final'])
    
    return best_result['x_final'], all_results

# 使用例：5つの初期値から試す
initial_values = [-1, 0, 2, 4, 6]

まとめ

今回学んだこと

  1. 谷が複数ある関数では、Newton法の結果が初期値に依存する
    • 左の谷の近くからスタート → 左の谷に落ちる
    • 右の谷の近くからスタート → 右の谷に落ちる
  2. ローカルミニマムとグローバルミニマム
    • ローカルミニマム = 周辺では一番低いが、全体では最低ではない
    • グローバルミニマム = 全体で本当に一番低い点
  3. 「収束した」≠「最適解を見つけた」
    • アルゴリズムは \(f'(x) = 0\) を見つけるだけ
    • それがローカルかグローバルかは判断できない
  4. 対策：複数の初期値から試して、一番良い結果を選ぶ
    • ランダムリスタートやグリッドサーチを使う
    • 2次微分で谷（最小値）かどうかも確認する
