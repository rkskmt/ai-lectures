今回の問題：体長から重さを予測する

  • 次元削減の回で測った体長を使う
  • 体長を測るだけで重さが分かれば、秤に乗せる手間が省ける

  • 直線でいけそう。直線のパラメータは傾き \(a\) と切片 \(b\) の2個
  • ではどの直線が「一番良い」？ — 良さを測る物差しが要る

損失関数：二乗誤差（MSE）

  • 各魚について「予測と実際のズレ」を測り、二乗して平均する

\[L(a, b) = \frac{1}{N} \sum_{i} \left( \underbrace{a x_i + b}_{\text{予測}} - \underbrace{t_i}_{\text{実際}} \right)^2\]

  • MSE（Mean Squared Error / 平均二乗誤差）と呼ぶ

クイズ

ズレの測り方なら絶対値でもいいはず。なぜ二乗？

  • 絶対値は 0 のところで微分できない（カクッと折れている）
  • 二乗なら微分がきれい — シグモイド＋交差エントロピーを選んだのと同じ理由
  • 勾配降下法を使う前提で、損失は「微分しやすいもの」を選ぶ

勾配を手で出す

\(L\) を \(a\) と \(b\) でそれぞれ微分する（合成関数の微分）：

\[\frac{\partial L}{\partial a} = \frac{2}{N} \sum_i (a x_i + b - t_i) \, x_i \qquad \frac{\partial L}{\partial b} = \frac{2}{N} \sum_i (a x_i + b - t_i)\]

  • 構造は分類のときと同じ：「予測と正解の差」を入力の大きさに応じて配る
  • あとは勾配降下法に渡すだけ：

\[a \leftarrow a - \eta \frac{\partial L}{\partial a}, \qquad b \leftarrow b - \eta \frac{\partial L}{\partial b}\]

勾配降下法で fit する

  • スケールを揃えてから（クラスタリングの回の教訓。学習率を1つで済ませるため）
x = (lengths - lengths.mean()) / lengths.std()   # 標準化した体長
t = (weights - weights.mean()) / weights.std()   # 標準化した重さ

a, b = 0.0, 0.0   # 直線はゼロからスタート
lr = 0.1          # 学習率

for i in range(1, 101):
    pred   = a * x + b
    grad_a = 2 * np.mean((pred - t) * x)
    grad_b = 2 * np.mean(pred - t)
    a -= lr * grad_a
    b -= lr * grad_b
    if i in (1, 3, 10, 50, 100):
        print(f"更新 {i:3d}回 | a={a:.4f} b={b:.4f} | MSE={np.mean((a*x+b - t)**2):.4f}")

モデルを複雑にしてみる

新しい仕入れ先の魚は15匹しかない

  • 別の仕入れ先から来た魚は、まだ15匹しか測れていない
  • 直線だけでなく、曲線も引ける：多項式回帰

\[\text{予測} = w_1 x + w_2 x^2 + \cdots + w_d x^d + b\]

  • \(x^2, x^3, \ldots\) を新しい特徴量として作れば、道具は線形回帰のまま
  • 次数 \(d\) がモデルの複雑さを決める

クイズ

次数を 1 → 3 → 9 と上げていくと、訓練誤差とテスト誤差はそれぞれどうなる？

次数を上げてみる

code 
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import PolynomialFeatures, StandardScaler
from sklearn.pipeline import make_pipeline
from sklearn.metrics import mean_squared_error

idx = np.random.default_rng(2).choice(80, 15, replace=False)   # 新しい仕入れ先の15匹
xs, ts = lengths[idx], weights[idx]
X_train, X_test, y_train, y_test = train_test_split(
    xs.reshape(-1, 1), ts, test_size=5, random_state=2
)

grid = np.linspace(xs.min() - 0.5, xs.max() + 0.5, 300).reshape(-1, 1)
fig, axes = plt.subplots(1, 3, figsize=(12, 3.6))
for ax, deg in zip(axes, [1, 3, 9]):
    model = make_pipeline(PolynomialFeatures(deg), StandardScaler(), LinearRegression())
    model.fit(X_train, y_train)
    ax.scatter(X_train, y_train, c="#2980b9", s=45, label="訓練")
    ax.scatter(X_test, y_test, c="#e74c3c", marker="s", s=45, label="テスト")
    ax.plot(grid, model.predict(grid), "k-", lw=2)
    ax.set_ylim(800, 2900)
    ax.set_title(f"次数 {deg}", fontsize=11)
    ax.set_xlabel("体長 (cm)")
axes[0].set_ylabel("重さ (g)"); axes[0].legend(fontsize=8)
plt.tight_layout()
plt.show()
  • 次数9は訓練の点をほぼ完璧に貫く。代わりに左端で大暴れ —「46.7cm の魚は 2900g 超」と主張している

数字で見る：また会ったな、過学習

degrees = range(1, 10)
train_mse, test_mse = [], []
for deg in degrees:
    model = make_pipeline(PolynomialFeatures(deg), StandardScaler(), LinearRegression())
    model.fit(X_train, y_train)
    train_mse.append(mean_squared_error(y_train, model.predict(X_train)))
    test_mse.append(mean_squared_error(y_test, model.predict(X_test)))

plt.figure(figsize=(6.5, 3.6))
plt.semilogy(degrees, train_mse, "o-", color="#2980b9", label="訓練MSE")
plt.semilogy(degrees, test_mse,  "s-", color="#e74c3c", label="テストMSE")
plt.xlabel("多項式の次数"); plt.ylabel("MSE（対数目盛）")
plt.legend(); plt.grid(True, alpha=0.4)
plt.tight_layout()
plt.show()
  • 訓練誤差は下がり続け、テスト誤差は桁違いに爆発（縦軸は対数）
  • 決定木の深さで見た過学習が、回帰でもまったく同じ形で現れた
  • 深さを制限したように、複雑さを抑える道具が要る — 回帰では正則化

正則化：大きい係数に罰金を払わせる

  • 暴れている曲線の正体は、巨大な係数
plain = make_pipeline(PolynomialFeatures(9), StandardScaler(), LinearRegression())
plain.fit(X_train, y_train)
print("次数9の係数の最大絶対値:", f"{np.abs(plain[-1].coef_).max():,.0f}")
次数9の係数の最大絶対値: 204,382
  • そこで、損失に「係数の大きさ」への罰金を足す：

\[L = \mathrm{MSE} + \alpha \sum_j w_j^2\]

  • これが Ridge 回帰（L2正則化）
  • 訓練データに合わせたい力と、係数を小さく保ちたい力の綱引き。\(\alpha\) がその強さ

Ridge で手なずける

from sklearn.linear_model import Ridge

fig, axes = plt.subplots(1, 2, figsize=(9.5, 3.6))
for ax, (model, name) in zip(axes, [
        (make_pipeline(PolynomialFeatures(9), StandardScaler(), LinearRegression()),
         "次数9（正則化なし）"),
        (make_pipeline(PolynomialFeatures(9), StandardScaler(), Ridge(alpha=1.0)),
         "次数9 + Ridge")]):
    model.fit(X_train, y_train)
    mse = mean_squared_error(y_test, model.predict(X_test))
    ax.scatter(X_train, y_train, c="#2980b9", s=45)
    ax.scatter(X_test, y_test, c="#e74c3c", marker="s", s=45)
    ax.plot(grid, model.predict(grid), "k-", lw=2)
    ax.set_ylim(800, 2900)
    ax.set_title(f"{name}　テストMSE {mse:,.0f}", fontsize=10)
    ax.set_xlabel("体長 (cm)")
    print(f"{name}: 係数の最大絶対値 = {np.abs(model[-1].coef_).max():,.0f}")
axes[0].set_ylabel("重さ (g)")
plt.tight_layout()
plt.show()
次数9（正則化なし）: 係数の最大絶対値 = 204,382
次数9 + Ridge: 係数の最大絶対値 = 74
  • 同じ次数9なのに、罰金を入れただけで素直な曲線に戻った
  • テストMSEは3桁以上改善

過学習スレッド、完結

引き（決定木のあやしい満点）→ 正体（訓練とテストのギャップ）→ 対抗手段（複雑さの制御と正則化）。 NN でも同じ道具（L2正則化＝weight decay）がそのまま使われる。

まとめ

  • 回帰：数値を予測する。損失は二乗誤差（MSE）— 微分しやすさで選ぶ
  • 勾配降下法で fit を自作 → sklearn と完全一致
  • 多項式回帰：特徴量に \(x^2, x^3, \ldots\) を足せば道具は線形回帰のまま
  • 過学習は回帰でも起きる：次数を上げると訓練誤差↓・テスト誤差爆発
  • 正則化（ridge）：大きい係数に罰金 → 暴れる曲線を手なずける
