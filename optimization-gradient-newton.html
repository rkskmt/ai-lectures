最適化問題とは

機械学習の中心課題

機械学習では、ほぼすべてのアルゴリズムで最適化問題を解く必要があります：

  • ロジスティック回帰: パラメータを調整して分類精度を最大化
  • ニューラルネットワーク: 重みを調整して予測誤差を最小化
  • サポートベクターマシン: マージンを最大化する境界を見つける

機械学習 = 最適化

「どうパラメータを調整すれば、最も良い結果が得られるか？」を数値的に解く

数学的には簡単ですが、今回は2つの数値的手法で解きます。

関数の定義

def f(x):
    """目的関数 f(x) = x^2 - 4x + 3"""
    return x**2 - 4*x + 3

def f_prime(x):
    """1次微分 f'(x) = 2x - 4"""
    return 2*x - 4

第1部：勾配降下法

勾配降下法とは

基本的な考え方

山登りの逆バージョン： 1. 現在地点での傾き（勾配）を測る 2. 傾きの逆方向に少し進む 3. また傾きを測る 4. 繰り返して最小値を探す

キーワード: 少しずつ、手探りで

アルゴリズム

更新式: \[x_{\text{new}} = x - \alpha \cdot f'(x)\]

  • \(x\): 現在の位置
  • \(f'(x)\): 勾配（傾き）
  • \(\alpha\): 学習率（どれくらい進むか）
  • \(x_{\text{new}}\): 次の位置

警告

学習率\(\alpha\)は自分で決める必要がある（これが難しい）

勾配降下法：手計算で実行

設定

  • 関数: \(f(x) = x^2 - 4x + 3\)
  • 1次微分: \(f'(x) = 2x - 4\)
  • 初期値: \(x_0 = 0\)
  • 学習率: \(\alpha = 0.3\)

反復1

ステップ1：現在の位置

\[x = 0\]

ステップ2：関数値を計算

\[f(0) = 0^2 - 4 \times 0 + 3 = 3\]

ステップ3：勾配を計算

\[f'(0) = 2 \times 0 - 4 = -4\]

意味: 傾きが\(-4\)（負）なので、右に行くほど値が小さくなる

ステップ4：更新

更新式: \[x_{\text{new}} = x - \alpha \cdot f'(x)\]

代入: \[x_{\text{new}} = 0 - 0.3 \times (-4)\]

計算: \[\begin{align} &= 0 - (-1.2) \\ &= 0 + 1.2 \\ &= 1.2 \end{align}\]

重要

反復1の結果: \(x = 0 \rightarrow x = 1.2\)

反復2

ステップ1：現在の位置

\[x = 1.2\]

ステップ2：関数値

\[f(1.2) = 1.2^2 - 4 \times 1.2 + 3 = 1.44 - 4.8 + 3 = -0.36\]

ステップ3：勾配

\[f'(1.2) = 2 \times 1.2 - 4 = 2.4 - 4 = -1.6\]

ステップ4：更新

\[x_{\text{new}} = 1.2 - 0.3 \times (-1.6) = 1.2 + 0.48 = 1.68\]

重要

反復2の結果: \(x = 1.2 \rightarrow x = 1.68\)

反復3

現在の位置

\[x = 1.68\]

勾配

\[f'(1.68) = 2 \times 1.68 - 4 = 3.36 - 4 = -0.64\]

更新

\[x_{\text{new}} = 1.68 - 0.3 \times (-0.64) = 1.68 + 0.192 = 1.872\]

反復4

現在の位置

\[x = 1.872\]

勾配

\[f'(1.872) = 2 \times 1.872 - 4 = 3.744 - 4 = -0.256\]

更新

\[x_{\text{new}} = 1.872 - 0.3 \times (-0.256) = 1.872 + 0.0768 = 1.9488\]

勾配降下法：Python実装

コードを表示 
# 初期設定
x = 0.0
alpha = 0.3  # 学習率

print("勾配降下法で最小値を求める")
print("=" * 60)

for iteration in range(1, 11):
    print(f"\n--- 反復 {iteration} ---")
    print(f"現在の x = {x:.4f}")
    
    # 関数値
    fx = f(x)
    print(f"f(x) = {fx:.4f}")
    
    # 勾配
    fp = f_prime(x)
    print(f"f'(x) = {fp:.4f}")
    
    # 更新
    step = alpha * fp
    x_new = x - step
    
    print(f"\n更新:")
    print(f"  ステップ幅 = α × f'(x) = {alpha} × {fp:.4f} = {step:.4f}")
    print(f"  x_new = {x:.4f} - {step:.4f} = {x_new:.4f}")
    
    x = x_new
    
    # 収束チェック
    if abs(fp) < 0.01:
        print(f"\n★ ほぼ収束（f'(x) < 0.01）")
        break

print("\n" + "=" * 60)
print(f"最終結果: x = {x:.4f}, f(x) = {f(x):.4f}")
print(f"理論値との誤差: {abs(x - 2.0):.6f}")

勾配降下法の問題点

1. 学習率の選択が難しい

警告

  • \(\alpha\)が小さすぎる: 収束が遅い
  • \(\alpha\)が大きすぎる: 振動して収束しない
  • 適切な\(\alpha\)を探すのが難しい

2. 収束が遅い

10回反復してもまだ誤差がある。複雑な問題では数百〜数千回必要になることも。

第2部：Newton法

なぜNewton法が必要か

勾配降下法の問題： - 学習率を手動で調整 - 収束が遅い

解決策: もっと賢く進めないか？

Newton法のアイデア

「傾き」だけでなく「曲がり具合」も使えば、どこまで進めばいいか計算できるのでは？

2次微分の役割

2次微分とは

1次微分: 傾き（どっちに進むか） \[f'(x) = 2x - 4\]

2次微分: 曲率（どれくらい曲がっているか） \[f''(x) = \frac{d}{dx}(2x - 4) = 2\]

2次微分の計算

1次微分をもう一度微分するだけ： \[f'(x) = 2x - 4\] \[f''(x) = 2 \times 1 - 0 = 2\]

ヘッセ行列（Hessian Matrix）とは

注意: 1変数の場合、2次微分は単なる数値なので「ヘッセ行列」とは呼びません。

ヘッセ行列は2変数以上で使う用語です：

2変数の場合（次回学びます）: \[H = \begin{bmatrix} \frac{\partial^2 f}{\partial x^2} & \frac{\partial^2 f}{\partial x \partial y} \\ \frac{\partial^2 f}{\partial y \partial x} & \frac{\partial^2 f}{\partial y^2} \end{bmatrix}\]

ノート

  • 1変数: \(f''(x)\) = 2次微分（スカラー）
  • 2変数以上: \(H\) = ヘッセ行列（行列）

今回は1変数なので「2次微分」と呼びます。

Newton法のアルゴリズム

更新式

\[x_{\text{new}} = x - \frac{f'(x)}{f''(x)}\]

  • \(f'(x)\): 1次微分（勾配）
  • \(f''(x)\): 2次微分（曲率）
  • 学習率不要！

なぜこの式？

現在地点で関数を2次関数で近似： \[f(x) \approx f(x_0) + f'(x_0)(x-x_0) + \frac{1}{2}f''(x_0)(x-x_0)^2\]

この2次関数の最小値を求めると： \[x_{\text{min}} = x_0 - \frac{f'(x_0)}{f''(x_0)}\]

重要

2次関数の最小値は解析的に求められる → 一気にそこまでジャンプできる！

Newton法：手計算で実行

設定

  • 関数: \(f(x) = x^2 - 4x + 3\)
  • 1次微分: \(f'(x) = 2x - 4\)
  • 2次微分: \(f''(x) = 2\)（重要！）
  • 初期値: \(x_0 = 0\)

反復1

ステップ1：現在の位置

\[x = 0\]

ステップ2：関数値

\[f(0) = 0^2 - 4 \times 0 + 3 = 3\]

ステップ3：1次微分（勾配）

\[f'(0) = 2 \times 0 - 4 = -4\]

ステップ4：2次微分（曲率）

\[f''(0) = 2\]

ステップ5：Newton法で更新

更新式: \[x_{\text{new}} = x - \frac{f'(x)}{f''(x)}\]

代入: \[x_{\text{new}} = 0 - \frac{-4}{2}\]

計算: \[\begin{align} &= 0 - (-2) \\ &= 0 + 2 \\ &= 2 \end{align}\]

重要

反復1の結果: \(x = 0 \rightarrow x = 2\)

反復2：収束確認

ステップ1：現在の位置

\[x = 2\]

ステップ2：関数値

\[f(2) = 2^2 - 4 \times 2 + 3 = 4 - 8 + 3 = -1\]

ステップ3：1次微分

\[f'(2) = 2 \times 2 - 4 = 4 - 4 = 0\]

収束！

勾配が0 → 極値に到達 → これ以上更新不要

答え

\[x = 2, \quad f(2) = -1\]

たった1回の反復で最適解に到達！

Newton法：Python実装

2次微分の定義

def f_double_prime(x):
    """2次微分 f''(x) = 2"""
    return 2

手で微分して関数定義

  • \(f(x) = x^2 - 4x + 3\)
  • \(f'(x) = 2x - 4\)
  • \(f''(x) = 2\)

手で計算した結果を関数として定義します。 自動微分は使いません。

Newton法の実装

コードを表示 
# 初期値
x = 0.0

print("Newton法で最小値を求める")
print("=" * 60)

for iteration in range(1, 6):
    print(f"\n--- 反復 {iteration} ---")
    print(f"現在の x = {x:.4f}")
    
    # 関数値
    fx = f(x)
    print(f"f(x) = {fx:.4f}")
    
    # 1次微分（勾配）
    fp = f_prime(x)
    print(f"f'(x) = {fp:.4f}")
    
    # 2次微分（曲率）
    fpp = f_double_prime(x)
    print(f"f''(x) = {fpp:.4f}")
    
    # Newton法の更新
    step = fp / fpp
    x_new = x - step
    
    print(f"\n更新:")
    print(f"  ステップ幅 = f'(x)/f''(x) = {fp:.4f}/{fpp:.4f} = {step:.4f}")
    print(f"  x_new = {x:.4f} - {step:.4f} = {x_new:.4f}")
    
    x = x_new
    
    # 収束チェック
    if abs(fp) < 0.0001:
        print(f"\n★ 収束！ f'(x) ≈ 0")
        break

print("\n" + "=" * 60)
print(f"最終結果: x = {x:.4f}, f(x) = {f(x):.4f}")

なぜNewton法は速いのか

視覚的説明

2次近似の威力

勾配降下法: 1次近似（接線）だけ → 「どっちに進むか」はわかるが「どこまで進むか」は手探り

Newton法: 2次近似（放物線） → 放物線の最小値を計算 → 一気にジャンプ

使い分けの指針

どちらを使うべきか

勾配降下法を使う場合: - パラメータ数が多い（数万〜数百万） - メモリが限られている - オンライン学習（データが逐次的に来る） - 例：ディープラーニング

Newton法を使う場合: - パラメータ数が少ない（数個〜数千） - 高精度な解が必要 - バッチ学習 - 例：ロジスティック回帰、一般化線形モデル

まとめ

勾配降下法

長所: - シンプルで実装が簡単 - 大規模問題に対応可能 - メモリ効率が良い

短所: - 学習率の調整が必要 - 収束が遅い

Newton法

長所: - 学習率不要（自動計算） - 収束が非常に速い - 高精度

短所: - 2次微分（ヘッセ行列）の計算が必要 - 大規模問題には不向き

ノート

どちらも重要！ 問題の性質に応じて使い分けることが大切です。

問題1の解答例

準備

\[f(x) = x^2 + 6x + 5\]

1次微分: \[f'(x) = 2x + 6\]

反復1

現在: \(x = 0\)

\[f'(0) = 2(0) + 6 = 6\]

更新: \[x_{\text{new}} = 0 - 0.2 \times 6 = 0 - 1.2 = -1.2\]

反復2

現在: \(x = -1.2\)

\[f'(-1.2) = 2(-1.2) + 6 = -2.4 + 6 = 3.6\]

更新: \[x_{\text{new}} = -1.2 - 0.2 \times 3.6 = -1.2 - 0.72 = -1.92\]

反復3

現在: \(x = -1.92\)

\[f'(-1.92) = 2(-1.92) + 6 = -3.84 + 6 = 2.16\]

更新: \[x_{\text{new}} = -1.92 - 0.2 \times 2.16 = -1.92 - 0.432 = -2.352\]

まだ収束していない（理論値は\(x=-3\)）
