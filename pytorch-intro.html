層の数や活性化関数を変えるたびに、backward を書き直すことになる。 100層のネットで、1ヶ所も間違えずに書ける？

  • 書けるかもしれないが、人間がやるべき仕事ではない
  • 連鎖律は機械的な作業 — 機械的な作業は機械にやらせる

PyTorch とは

  • ディープラーニングの業界標準ライブラリのひとつ
  • 今日使う機能は実質2つだけ
    • autograd：backward を自動でやってくれる

ノート

この講義の環境にはインストール済み（environment.yml の pytorch-cpu / torchvision）。 自分のPCなら pip install torch 等で入る。

tensor：NumPy 配列の親戚

import torch
import numpy as np

t = torch.tensor([1.0, 2.0, 3.0])
print(t * 2)          # 演算は NumPy とほぼ同じ感覚
print(t.numpy())      # NumPy 配列との相互変換も簡単
tensor([2., 4., 6.])
[1. 2. 3.]
  • 違いは2つ：GPU に載せられること、そして勾配を覚えられること
a = torch.tensor([0.3, -0.2], requires_grad=True)   # 「勾配を追跡せよ」の印
print(a)
tensor([ 0.3000, -0.2000], requires_grad=True)
  • requires_grad=True を付けた tensor は、自分が関わった計算をすべて記録する

山場：autograd が手計算を言い当てる

  • バックプロパゲーションの回とまったく同じネットワーク・同じ数値を流す
a = torch.tensor([0.3, -0.2], requires_grad=True)   # α1, α2
b = torch.tensor(0.1,  requires_grad=True)
w = torch.tensor(0.5,  requires_grad=True)
c = torch.tensor(-0.1, requires_grad=True)
x = torch.tensor([1.0, 0.5])                        # 入力（正解 y=1）

z = a @ x + b          # アフィン変換
h = torch.relu(z)      # ReLU
u = w * h + c          # 出力層
p = torch.sigmoid(u)   # シグモイド
L = -torch.log(p)      # 交差エントロピー（正解1）

L.backward()           # ← 復路はこの1行

print(f"L = {L.item():.4f}")
print(f"∂L/∂α = {a.grad.numpy().round(4)}")
print(f"∂L/∂b = {b.grad.item():.4f}")
print(f"∂L/∂w = {w.grad.item():.4f}")
print(f"∂L/∂c = {c.grad.item():.4f}")
L = 0.6685
∂L/∂α = [-0.2438 -0.1219]
∂L/∂b = -0.2438
∂L/∂w = -0.1463
∂L/∂c = -0.4875

答え合わせ

手計算（とNumPy検算）の答えは α: \((-0.2438, -0.1219)\)、b: \(-0.2438\)、w: \(-0.1463\)、c: \(-0.4875\)。 全桁一致。 loss.backward() の中身は、前回自分の手でやった連鎖律の機械化にすぎない。

なぜ自動でできるのか

  • forward の間、PyTorch は計算の履歴（計算グラフ）を記録している
  • backward() は、その履歴を逆順にたどって連鎖律を適用するだけ
  • やっていることは前回の手計算1往復と同じ：
    • 往路の途中の値を保存（だからメモリを食う）
    • \(\delta\) を出力側から掛けて貯める

ヒント

ブラックボックスとして使うのと、中身を知って使うのは別物。 われわれは中身を手で再現済みなので、堂々と楽をしてよい。

nn：部品には全部、見覚えがある

import torch.nn as nn

model = nn.Sequential(
    nn.Linear(2, 16),   # アフィン変換 X @ W1 + b1（パラメータは自動で用意・初期化）
    nn.ReLU(),          # 折る
    nn.Linear(16, 1),   # 出力層 H @ w2 + c2
)
print(model)
Sequential(
  (0): Linear(in_features=2, out_features=16, bias=True)
  (1): ReLU()
  (2): Linear(in_features=16, out_features=1, bias=True)
)
PyTorch の名前 この講義で作った部品
nn.Linear アフィン変換（X @ W1 + b1）
nn.ReLU \(\max(0, z)\)
nn.BCEWithLogitsLoss シグモイド＋交差エントロピー（打ち消しもやってくれる）
torch.optim.SGD 勾配降下法（θ -= lr * grad）
loss.backward() バックプロパゲーション
  • 新しい概念はゼロ。全部、名前が付いただけ

学習ループ：3拍子

from sklearn.datasets import make_moons

X, y = make_moons(n_samples=200, noise=0.1, random_state=42)
X_t = torch.tensor(X, dtype=torch.float32)
y_t = torch.tensor(y, dtype=torch.float32).unsqueeze(1)   # (200,) → (200,1)

torch.manual_seed(0)
model = nn.Sequential(nn.Linear(2, 16), nn.ReLU(), nn.Linear(16, 1))
loss_fn   = nn.BCEWithLogitsLoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.5)

for i in range(1, 4001):
    optimizer.zero_grad()        # ① 前回の勾配をリセット
    logits = model(X_t)
    loss = loss_fn(logits, y_t)
    loss.backward()              # ② backprop（自動）
    optimizer.step()             # ③ 勾配降下で1歩

    if i in (1, 100, 500, 2000, 4000):
        acc = ((torch.sigmoid(logits) >= 0.5).float() == y_t).float().mean()
        print(f"更新 {i:5d}回 | Loss: {loss.item():.4f} | 正答率: {acc.item():.1%}")
更新     1回 | Loss: 0.6807 | 正答率: 50.0%
更新   100回 | Loss: 0.2652 | 正答率: 85.5%
更新   500回 | Loss: 0.0449 | 正答率: 99.0%
更新  2000回 | Loss: 0.0057 | 正答率: 100.0%
更新  4000回 | Loss: 0.0024 | 正答率: 100.0%
  • NumPy 版と同じく 100%。zero_grad → backward → step の3拍子はどんなネットでも同じ
