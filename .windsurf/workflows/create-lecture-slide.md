# AI講義スライド作成ガイドライン

## 対象読者

- **高校生**でもわかるレベルを目指す
- 前提知識は中学数学＋前回の講義内容のみ

## ファイル形式

- Quarto (.qmd) で作成
- プロジェクト: `/home/rkskmt/docs/ai/`
- 設定ファイル: `_quarto.yml`

## 構成の原則

### 体験→発見→概念→対策の流れ

1. **前回の振り返り** — 前回の内容を簡潔に復習し、今回の問題につなげる
2. **具体例の提示** — グラフや関数を見せて「何が起きるか」を予想させる
3. **手計算で体験** — 実際に手で計算を追いかけて、現象を体験する
4. **結果の発見** — 「あれ、こうなるんだ！」という気づきを促す
5. **概念に名前をつける** — 体験した後に用語・定義を導入する（先に用語を出さない）
6. **対策・応用** — 問題の解決策を、まず手計算で、次にコードで示す

### やってはいけないこと

- 用語を定義してから体験させる（順序が逆）
- コードだけ見せて「結果はこうなります」で済ませる
- 前提知識なしに専門用語を使う

## 本文（Markdown部分）

- **日本語**で書く
- 見出し階層: `#` → `##` → `###` → `####`
- セクション間は `---`（水平線）で区切る

### Calloutブロック

Quartoのcalloutを活用する。**1つのcallout内に `##` 見出しは1つだけ**（複数入れるとレンダリングが壊れる）。

```markdown
::: {.callout-note}
## タイトル（1つだけ）
内容
:::
```

使い分け:
- `.callout-note` — 補足説明、観察ポイント
- `.callout-tip` — コツ、覚えておくべきこと
- `.callout-important` — 重要な結果、チェックリスト項目
- `.callout-warning` — 注意点、よくある間違い

## Pythonコードブロック

### 基本構造

```markdown
｀｀｀{python}
#| echo: true
# コード
｀｀｀
```

### 表示制御オプション

| オプション | 用途 |
|---|---|
| `#\| echo: true` | コードを表示する（学習用） |
| `#\| echo: false` | コードを非表示にする（グラフ生成用） |
| `#\| code-fold: true` | 折りたたみにする（長いコード） |
| `#\| code-summary: "コードを表示"` | 折りたたみのラベル |
| `#\| fig-align: center` | グラフを中央揃え |

### コード内のコメント

- コード内のコメントは**日本語OK**（コメントはmatplotlibで描画されないため）

## matplotlib（グラフ）のルール

### ★最重要：plt内のテキストはすべて英語

matplotlibのデフォルトフォントは日本語非対応のため、**plt関数に渡す文字列はすべて英語**にする。

対象:
- `plt.title()` / `ax.set_title()`
- `plt.xlabel()` / `ax.set_xlabel()`
- `plt.ylabel()` / `ax.set_ylabel()`
- `label=` パラメータ（凡例に表示される）
- `plt.annotate()` のテキスト
- `Patch(label=...)` など凡例要素

```python
# ✗ NG: 文字化けする
plt.title('4次関数のグラフ：谷が2つある！')
plt.xlabel('スタート地点')
label='ローカルミニマム'

# ✓ OK: 英語で書く
plt.title('Quartic Function: Two Valleys!')
plt.xlabel('Starting Point')
label='Local Min'
```

### グラフのスタイル

- `figsize`: 横長 `(12, 5)` 〜 `(14, 6)` が基本
- `fontsize`: タイトル14、軸ラベル12-13、凡例10-11
- `grid=True, alpha=0.3`
- `plt.tight_layout()` を必ず入れる
- 色の使い分け:
  - 赤 `'r'` — 注意・ローカル・悪い結果
  - 緑 `'g'` — 良い結果・グローバル
  - 青 `'b'` — 関数本体・中立

## 手計算パートの書き方

### 構造

```markdown
#### 反復1

**現在の位置**: $x = 1.5$

**1次微分（傾き）を計算:**
$$f'(1.5) = 0.4 \times 1.5^3 - 3 \times 1.5^2 + 4 \times 1.5$$
$$= 1.35 - 6.75 + 6.0 = 0.6$$

**更新:**
$$x_{\text{new}} = 1.5 - \frac{0.6}{-2.3} = 1.761$$
```

### ポイント

- 途中式を省略しない（高校生が追えるように）
- 各ステップに「何を計算しているか」のラベルをつける
- 手計算の後に `Pythonで確認` セクションを置き、コードで検算する

## 数式

- インライン: `$f'(x) = 0$`
- ブロック: `$$f(x) = 0.1x^4 - x^3 + 2x^2 + 1$$`
- 変数名は一貫させる（$x_0$, $x_{\text{new}}$, $f'(x)$, $f''(x)$）
